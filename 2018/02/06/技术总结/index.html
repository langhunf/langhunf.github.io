技术总结

###ajax 介绍

* ajax是一种技术方案，但并不是一种新技术，他依赖现有的css/html/javascript,而其中最核心的
* 依赖的是浏览器提供的XMLHttpRequesr对象，是这个对象使得浏览器可以发出HTTP请求接收HTTP响应
* 
* ajax:(异步js和XHL)
* 同步js:js的代码都是从上到下执行的，如果一块代码没有执行完毕，后面的代码不会执行
* 异步就是ajax可以不按顺序执行
* 
* XHL：存储数据的一种格式
* 
* ajax不光处理XHL格式的数据，还可以处理JSON数组字符串类型的数据
* js与后端进行数据交互的一种技术，通过请求协商好的接口，来获取想要的数据
* 
* ajax是基于HTTP协议，只能在HTTP服务器下运行
* 本地协议file
* 
* www目录下的文件夹不能以中文命名，但是文件里的HTML文件名可以用中文命名
* localhost相当于www的目录
* 
* 优点：
* 传输数据时会在本页面请求服务器，不用跳转页面，从而减轻服务器的压力，做到实时验证，优化用户体验
* 
* ajax 是运行在服务器上的
* name 是用来提交数据的
* 
* action    表单提交的地址    取name的值
* method    就是提交的方式
*         get 提交数据通过地址栏发送，数据添加到地址栏的问号后面
*             数据形式：数据名=数据值&数据名=数据值
*         post通过请求头发送(头部信息，能通过控制台看到)
* 
* XHR ajax的一个对象

###ajax 数据交互流程

1.创建一个ajax对象    
 2.填写请求地址
 3.发送请求
 4.等待服务器响应
 5.接收数据

 *XMLHttpRequest 这个对象身上有一个open()方法
 *第一个参数    请求方式
 *第二个参数    地址
 *第三个参数    同步还是异步        true表示异步请求


window.onload=function(){
    var inputs=document.querySelectorAll('input');
    inputs[2].onclick=function(){//点击的时候发送请求
        var val=inputs[0].value;//请求内容
        //创建一个ajax对象
        var ajax=new XMLHttpRequest;
        //填写请求地址
        ajax.open('get','php/get.php?user='+val,true);
        //发送请求
        ajax.send();
        //等待服务器响应
        ajax.onload=function(){
      //接收数据     响应的都在new XMLHttpRequest.responseTEXT
            var span=document.querySelector('span');
            span.innerHTML=ajax.responseText;
        }
    }
}



 * ajax.readState    ajax运行步骤(第一步是捕捉不到的)
 * 它的值为4的时候说明ajax运行完成
 * 0 初始化    还没有调用open()方法
 * 1载入        已经调用open()方法，正在发送请求
 * 2载入完成    send()方法完成，已经收到全部响应内容
 * 3解析        正在解析响应内容
 * 4完成        响应内容解析完成，可以在客户端使用了
 * 
 * 服务器
 * ajax.status    服务器对请求的反馈(状态码)
 * 200    成功
 * 404    请求资源没有找到
 * 505    服务器错误
 * 
 * onreadystatechange
 *     readState的值发生变化的时候触发的事件，只要这个值有变化就会触发
 * onload
 *     所有的请求成功后触发，此时readState的值为4(ie6 7 8 不支持)
 * 
 * 新的XHLHttpRequest不推荐使用onreadystatechange，推荐onload
 *

###get post区别介绍

######get
 1.通过地址栏传输数据，传输的大小有限制
 ajax.com('get','php/get.php?user='+val,true)
 2.不安全，用户的所有信息全部会暴露
 3.拼接数据的时候要用encodeURI转码，不然中文就会乱码
 4.不用设置请求头
 5.数据的拼接在open方法里
 6.会有缓存问题

######post
1.通过send向服务器传输数据，理论上来说是没有或者体积的限制
 ajax.send()
2.相对安全，不暴露用户信息
3.不用转码，已经通过请求头设置了数据格式，不会有乱码
 ajax.setRequesHeader('Content-Type','application/x-www-form-urlencoded')
4.在send()的前面需要设置一个请求头
5.数据拼接的时候要在send方法里边
6.没有缓存问题

###什么叫跨域及他的解决办法

* 跨域
*   两个不同域名下的数据进行交互，Ajax之所以不能跨域其实是因为XMLHttpRequest受到同源策略的限制，只能让它访问同源下的数据，不能访问不同源下的数据
*   同源策略
*      每个网站只能访问同一来源的数据，同一来源是指主机名(域名)。协议(http/https)和端口号的组合，在没有明确授权的情况下，不能读写对方资源，它是浏览器最核心的也是最基本的安全功能
*   源：协议、端口、域名
*       只要不同源就跨域
* 协议
*   http  超文本协议
*   https HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，比http协议安全
*   file  本地协议
*   ftp   文件共享协议
* 端口
*   相当于酒店里一层的房间
* 域名
*   https://www.kaivon.cn/
* 
* http://www.kaivon.cn/c/d.html 成功
* http://www.kaivon.cn/d/e/f.html 成功
* https://www.kaivon.cn/a/c.html  失败 协议不同
* http://www.kaivon.cn:81/a/c.html  失败 端口不同
* http://www.apeclass.com/a/b.html  失败 主机名不同
* 
* 跨域解决方法
*   1.在标准浏览器下XMLHttpRequest配合后端设置一个请求权限
*       在php里写上  header('Access-Control-Allow-Origin:*');
*       这个是XMLHttpRequest 2级支持的，非标准浏览器不支持
*   2.服务器代理
*        服务器文件可以访问你想要的资源，而你又能访问服务器文件，所以就可以访问想要的资源
*        缺点：如果想访问的资源需求有变化，对后端的开发成本就很大
*   3.iframe、flash、postMessage、WebSocket
*        http://www.freebuf.com/articles/web/65468.html
*   
* 主流解决方法  : 带src属性的<script><img><link>等标签是不需要遵守同源策略的，但是通过src加载的资源，浏览器限制了JS的权限，只能读不能写
*   4.jsonp(json+padding)
*    1.通过script标签引入某些数据，是(同步模式)的，只有上面的代码加载完成才能继续加载下面的代码，
*       一般在用script标签做跨域时，不建议将数据提前加载，应该按需加载
*    2.当需要数据的时候创建一个script标签，将需要的数据放在src中，通过onload去监听是否请求过来
*       请求完毕就调用传回来的数据，(异步加载)
*    3.jsonp不能用post请求，只能用get方式请求

###同步和异步
>
同步
1.当send方法调用后会等待服务器返回信息，如果服务器一直没有响应，就会阻塞后面的代码
2.后面的代码执行受前面代码的影响
异步
1.当send方法调用后，就会执行后面的代码，不用等待服务器响应
2.后面的代码不受前面代码的影响
